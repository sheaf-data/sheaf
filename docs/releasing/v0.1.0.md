# sheaf v0.1.0

Initial public release. Sheaf maps a project's real contract surface —
every CLI command, API method, and config knob — and shows, element by
element, which ones a doc explains, a test verifies, and a working example
demonstrates. The empty cells are where coding agents guess; sheaf makes
them visible so agents can ground on the contract instead.

## Highlights

- **`sheaf` CLI** — scan a project's contract surface and report
  documentation-, test-, and example-coverage gaps. Subcommands: `init`,
  `doctor`, `scan`, `snapshot`, `serve`, `report`, `gaps`, `coverage`,
  `review`, `version`, plus monorepo fan-out via `sheaf scan --manifest`.
- **MCP server** (`sheaf serve`) — exposes the in-memory contract index
  over JSON-RPC so coding agents can ground answers in the real surface,
  tests, and docs. Configurable bind/port/auth via `mcp_server { … }`.
- **Self-contained HTML reports** — worklist, coverage matrix, and findings
  (tested-undocumented, thin-reference, stale-doc, …) rendered to one file,
  with the `scanner` companion binary and the `sheaf review` PR-comment bot.
- **Broad format coverage out of the box** — contract anchors (`fidl`,
  `proto`, `cobra`, `argh`, `clap`, `cml`, `cppheader`), tests (`gotest`,
  `gtest`, `rusttest`, `bats`, `pytest`, `protocpp`), docs (`markdown`,
  `rst`, `workflows`, `yamlworkflows`), and rendered references (`fidldoc`,
  `clidoc`, `markdowncli`).
